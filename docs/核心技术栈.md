# YTBT 核心技术栈说明

**YTBT**（YouTube / 视频字幕工作台）是一款在浏览器中运行的**个人字幕制作工具**：从视频或音频完成语音识别（ASR）、时间轴处理、中文翻译，并导出 SRT。媒体与密钥处理以**本机浏览器**为主，云端 API 仅用于 ASR 与 LLM 翻译。

---

## 1. 技术栈总览

| 层次 | 技术 | 作用 |
|------|------|------|
| 界面框架 | **Vue 3**（Composition API + `<script setup>`） | 单页应用 UI |
| 路由 | **Vue Router 4** | 三个业务页面与懒加载 |
| 状态 | **Pinia 3** | 各页 Store + 全局 ASR 设置 |
| UI 组件库 | **Element Plus 2** + **@element-plus/icons-vue** | 表格、表单、进度、消息等 |
| 语言 | **TypeScript 6** | 类型安全 |
| 构建 | **Vite 5** + **vue-tsc** | 开发服务器、生产打包、类型检查 |
| 样式 | **Sass（SCSS）** | 全局与组件 scoped 样式 |
| 本机音视频 | **@ffmpeg/ffmpeg**（WebAssembly） | 抽音轨、按轴切音、拼接 |
| 本机存储 | **IndexedDB** + **localStorage** | 转写缓存、工作台草稿、UI 参数 |
| 云端 ASR | **Replicate API**（WhisperX）/ **Deepgram API**（Nova） | 语音转文字与时间戳 |
| 云端翻译 | **OpenAI 兼容 Chat API**（默认 JieKou AI） | 字幕批量译为中文 |

```text
┌─────────────────────────────────────────────────────────────┐
│                     浏览器（Vue 3 SPA）                      │
│  Views / Components  ←→  Pinia Stores  ←→  Utils / Types   │
├─────────────────────────────────────────────────────────────┤
│  FFmpeg.wasm（本机）  │  IndexedDB / localStorage（本机）    │
├─────────────────────────────────────────────────────────────┤
│  Vite Dev Proxy：/replicate  /deepgram  /openai            │
└──────────────┬──────────────────┬──────────────────┬──────┘
               ▼                  ▼                  ▼
         Replicate            Deepgram          LLM（JieKou 等）
         (WhisperX)           (Nova-2/3)
```

## 2. 前端核心

### 2.1 Vue 3 + TypeScript

- 入口：`src/main.ts`，挂载 `App.vue`，注册 Pinia、Router、Element Plus（中文 locale）。
- 组件以 **Composition API** 与 **`<script setup>`** 为主，逻辑按页面拆到 `stores/` 与 `services/`。
- 路径别名：`@/` → `src/`（在 `vite.config.ts` 中配置）。

### 2.2 Vue Router

| 路径 | 页面 | 职责 |
|------|------|------|
| `/` | `Workbench.vue` | 视频转写 → 切行 → 翻译 → 导出 |
| `/auto-timing` | `AutoTiming.vue` | 词级 ASR → 自动打轴（默认每行≤18 字） |
| `/timed-asr` | `TimedAsr.vue` | 导入已有时间轴 → 按轴填词 → 翻译 |

路由使用 **History 模式**，页面组件 **动态 import** 以减小首包体积。

### 2.3 Pinia 状态

| Store | 文件 | 说明 |
|-------|------|------|
| `workbench` | `stores/workbench.ts` | 主工作台：媒体、字幕段、转写/翻译流程、草稿恢复 |
| `asrSettings` | `stores/asrSettings.ts` | 全局 ASR 提供商（WhisperX / Deepgram）及 Nova 选项 |
| `autoTiming` | `stores/autoTiming.ts` | 自动打轴参数、词级单元缓存、跳转填词页 |
| `timedAsr` | `stores/timedAsr.ts` | 轴先导填词、切音拼接 / 全片映射 |

跨页 ASR 模型选择在顶栏 `AppNav.vue` 中绑定 `asrSettings`。

### 2.4 Element Plus

用于工具栏、表格（`SubtitleTable`）、上传、下拉导出、数字输入、告警与进度条等。图标来自 `@element-plus/icons-vue`。

## 3. 构建与工程化

### 3.1 Vite 5

- **开发**：`npm run dev`，HMR，环境变量从 `.env.local` 读取（前缀 `VITE_`）。
- **生产**：`npm run build` → `vue-tsc -b` 类型检查 + `vite build` 输出到 `dist/`。
- **预览**：`npm run preview`。

### 3.2 开发时代理（避免 CORS、隐藏密钥）

在 `vite.config.ts` 的 `server.proxy` 中：

| 前端路径 | 上游 | 用途 |
|----------|------|------|
| `/replicate/*` | `https://api.replicate.com` | WhisperX 上传与 Prediction |
| `/deepgram/*` | `https://api.deepgram.com` | Deepgram 预录转写 |
| `/openai/*` | `VITE_LLM_API_BASE_URL`（默认 JieKou） | Chat Completions 翻译 |

代理在请求头注入 `Authorization`，**密钥只存在于本机 `.env.local`**，不写入前端打包产物（但仍会进入开发环境内存，勿将 `.env.local` 提交 Git）。

### 3.3 TypeScript 与样式

- `typescript` ~6.0，`vue-tsc` 做 Vue SFC 类型检查。
- `sass` 编译 `src/styles/main.scss` 与各组件 `lang="scss"`。

### 3.4 可选插件说明

`package.json` 中含 `unplugin-auto-import`、`unplugin-vue-components`，当前入口以显式 import 为主；若未在 `vite.config` 启用，则不影响运行。

## 4. 本机能力（浏览器内）

### 4.1 FFmpeg WebAssembly

- 包：`@ffmpeg/ffmpeg`、`@ffmpeg/util`。
- 封装：`src/services/ffmpegClient.ts`（单例加载 core/wasm）。
- 用途：
  - `audioExtract.ts`：从视频提取音轨，减小上传体积；
  - `timelineConcatAudio.ts`：按字幕行切音、静音间隔、拼接后一次送 ASR（轴先导「切音拼接」模式）。

Vite 将 `@ffmpeg/ffmpeg` 排除在 `optimizeDeps` 之外，避免预打包问题。

### 4.2 IndexedDB 缓存

- 库名：`ytbt-cache`（`src/services/cache/idb.ts`）。
- 存储：媒体指纹、转写结果、翻译结果、元数据。
- 上层：`subtitleCache.ts`、`draft.ts`、`fingerprint.ts` 根据文件哈希与 ASR/切分参数生成缓存键，避免重复调用 API。

**说明**：自动打轴页、轴先导页目前主要用 `localStorage` 存 UI 参数；工作台支持完整草稿恢复。

### 4.3 媒体与字幕解析

- 视频预览：`<video>` + `URL.createObjectURL`（`VideoPlayer.vue`）。
- 字幕格式：`parseSrt.ts`、`parseAss.ts`、`parseTimeline.ts`；导出 `utils/srt.ts`（含空轴导出）。

## 5. 云端服务与 AI 管线

### 5.1 统一转写入口

`src/services/transcription.ts` 的 `runTranscription()` 根据 `asrSettings.provider` 分支：

1. **WhisperX（Replicate）**  
   - `replicate.ts`：上传文件 → `POST /v1/predictions`（固定 model version）→ 轮询结果。  
   - 支持 VAD、词级对齐、说话人分离（需 HuggingFace Token）等。  
   - 解析：`whisperxParser.ts` → `extractTimedUnits()` 得到词/段级时间单元。

2. **Deepgram（Nova-2 / Nova-3）**  
   - `deepgram.ts`：预录 listen API。  
   - 解析：`deepgramParser.ts`，输出与 WhisperX 相近的 segment 结构以便复用后续逻辑。

### 5.2 字幕后处理（本机算法）

| 模块 | 作用 |
|------|------|
| `segmentSplit.ts` | 按字数、标点、最长时长切分过长段（工作台） |
| `buildTimelineFromUnits.ts` | 词间停顿 + 每行字数上限 → 生成时间轴（自动打轴） |
| `mapAsrToTimeline.ts` | 将 ASR 结果映射到已有时间轴行（轴先导填词） |
| `cjkText.ts` | CJK 去多余空格等 |

### 5.3 翻译

- `translate.ts`：使用 **OpenAI 官方 SDK**（`openai` 包），`baseURL` 指向 `/openai`，兼容 JieKou 等 OpenAI 格式网关。
- 批量翻译字幕 `text` 字段，写入 `translatedText`。

## 6. 三条业务流水线（概念）

### 6.1 视频转写（工作台）

```text
媒体 → [可选 FFmpeg 抽音] → Replicate/Deepgram ASR
     → parse + splitLongSegments → 字幕表 → LLM 翻译 → 导出 SRT
```

### 6.2 自动打轴

```text
媒体 → ASR（必须词级时间戳）→ buildTimelineFromUnits（默认≤18 字/行）
     → 预览 / 导出空轴 / sessionStorage 跳转轴先导填词
```

### 6.3 轴先导填词

```text
媒体 + 已有 SRT/ASS 时间轴 → [可选按行切音拼接] → ASR → 按时间映射填字
     → 翻译 → 导出
```

复制 `.env.example` 为 `.env.local` 后配置：

| 变量 | 必需场景 | 说明 |
|------|----------|------|

## 9. 设计特点与边界

**特点**

- **纯前端 SPA**：无自建后端；部署静态 `dist/` 即可（开发期依赖 Vite 代理）。
- **ASR 可切换**：WhisperX 与 Deepgram 共用后续处理链路。
- **轴与字分离**：自动打轴 / 轴先导填词针对「时间轴不准」场景，与工作台「一条龙转写」互补。
- **本机减负**：FFmpeg 抽音、切音拼接降低上传量；IndexedDB 降低重复转写成本。

**边界**

- 大文件受 Replicate / Deepgram 与浏览器内存限制；超大视频建议先抽音轨。
- API 密钥在开发环境由代理使用，生产若静态部署需另行解决跨域与鉴权（例如自建反向代理）。
- 轴质量依赖词级时间戳与停顿规则，强 BGM / 多人重叠场景仍需 Aegisub 等工具微调。
